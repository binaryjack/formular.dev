Current Button Styling Issues

Current Button Issues

Expected: Solid Primary (gray bg, white text)

Expected: Outlined Primary (transparent bg, gray text, gray border)

Expected: Solid Danger (red bg, white text)

Expected: Outlined Danger (transparent bg, red text, red border)

Identified Issues

Button Component Issues:

  • Array handling: Using cx(styles.backGround) instead of spreading the array
  • Missing styles: Not applying border styles or state styles (hover, focus, etc.)
  • Typography integration: Passing CSS classes as string instead of variant object
  • Incomplete styling: Only applying background, missing text and border classes

Required Fixes

// Current problematic code:
className={`${cx(styles.backGround)} relative overflow-hidden`}

// Should be:
className={cx(
    styles.backGround,
    styles.borders,
    ...Object.values(styles.states),
    'relative', 
    'overflow-hidden'
)}

// Typography fix:
// Current:
variants={`${cx(styles.text)} text-nowrap elipsed-text`}

// Should be:
className={cx(styles.text, 'text-nowrap', 'elipsed-text')}
variants={variants} // Pass the variant object, not CSS classes
